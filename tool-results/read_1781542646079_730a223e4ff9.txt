     1в†’# PRD вЂ” Kiwul Storyboard Studio

     7в†’
     8в†’---
     9в†’
    10в†’## 1. Ringkasan Produk
    11в†’
    12в†’**Kiwul Storyboard Studio** adalah program khusus untuk mengeksekusi storyboard berbasis JSON. Program ini tidak berfokus pada ideation atau pembuatan script dari nol, tetapi fokus pada tahap produksi visual: membaca storyboard dari JSON, menampilkan setiap scene, mengirim prompt image dan video ke provider lokal, lalu menyimpan serta menyediakan hasil image/video untuk di-preview dan di-download.
    13в†’
    14в†’Setiap scene memiliki tiga komponen utama:
    15в†’
    16в†’1. **VO** вЂ” narasi atau voice over.
    17в†’2. **Image Prompt** вЂ” prompt untuk membuat gambar scene.
    18в†’3. **Video Prompt** вЂ” prompt untuk membuat video dari gambar scene.
    19в†’
    20в†’Program harus mendukung aspect ratio `9:16` dan `16:9`, durasi total dari `30 detik` sampai `15 menit`, jumlah scene mengikuti isi JSON, serta tombol **Generate All** dengan sistem queue dan polling sehingga hasil scene yang selesai langsung muncul tanpa menunggu seluruh proses selesai.
    21в†’
    22в†’---
    23в†’
    24в†’## 2. Tujuan Produk
    25в†’
    26в†’Tujuan Kiwul Storyboard Studio:
    27в†’
    28в†’1. Membaca storyboard dari file JSON.
    29в†’2. Menampilkan scene dalam UI yang mudah dipahami.
    30в†’3. Menyediakan editor VO, image prompt, dan video prompt per scene.
    31в†’4. Menghubungkan program ke provider image dan video.
    32в†’5. Generate image per scene.
    33в†’6. Generate video per scene.
    34в†’7. Generate semua scene melalui tombol **Generate All**.
    35в†’8. Memakai sistem queue dan polling.
    36в†’9. Menampilkan hasil image/video langsung saat selesai.
    37в†’10. Menyediakan tombol download image dan video.
    38в†’11. Menyediakan export JSON final berisi path hasil generate.
    39в†’12. Tidak hardcoded pada provider, model, aspect ratio, durasi, atau jumlah scene.
    40в†’

    42в†’
    43в†’## 3. Masalah yang Diselesaikan
    44в†’
    45в†’Masalah utama dalam produksi video AI lokal:
    46в†’
    47в†’1. Storyboard sering hanya berupa teks atau JSON yang belum bisa langsung dieksekusi.
    48в†’2. VO, image prompt, dan video prompt sering tidak sinkron.
    49в†’3. User harus copy-paste prompt satu per satu ke engine image/video.
    50в†’4. Hasil image/video sulit dikaitkan kembali ke scene.
    51в†’5. Tidak ada status per scene.
    52в†’6. Jika satu scene gagal, user sulit regenerate hanya scene itu.
    53в†’7. Tidak ada tombol download per scene.
    54в†’8. Tidak ada sistem batch generation yang aman untuk GPU lokal.
    55в†’9. Tidak ada polling progress sehingga user tidak tahu proses sedang berjalan atau macet.
    56в†’
    57в†’Kiwul Storyboard Studio menyelesaikan masalah ini dengan satu workspace produksi storyboard yang terstruktur.

  1226в†’## 32. Validasi JSON
  1227в†’
  1228в†’Saat load JSON, sistem wajib mengecek:
  1229в†’
  1230в†’1. JSON valid.
  1231в†’2. `project` ada.
  1232в†’3. `scenes` ada dan berupa array.
  1233в†’4. Minimal 1 scene.
  1234в†’5. Setiap scene punya `scene_id`.
  1235в†’6. Setiap scene punya `vo`.
  1236в†’7. Setiap scene punya `image_prompt`.
  1237в†’8. Setiap scene punya `video_prompt`.
  1238в†’9. Setiap scene punya `duration`.
  1239в†’10. Tidak ada `scene_id` duplikat.
  1240в†’11. `aspect_ratio` hanya `9:16` atau `16:9`.
  1241в†’12. Total durasi antara `30` dan `900` detik.
  1242в†’13. Jika `duration_seconds` tidak cocok dengan total scene, tampilkan warning.
