     1→     1→# PRD — Kiwul Storyboard Studio
     2→     2→
     3→     3→**Versi:** 1.0  
     4→     4→**Status:** Draft lengkap  
     5→     5→**Produk:** Modul Storyboard mandiri dari Kiwul  
     6→     6→**Tujuan utama:** Load JSON storyboard → tampilkan VO, image prompt, video prompt → generate image/video → preview → download hasil.
     7→     7→
     8→     8→---
     9→     9→
    10→    10→## 1. Ringkasan Produk
    11→    11→
    12→    12→**Kiwul Storyboard Studio** adalah program khusus untuk mengeksekusi storyboard berbasis JSON. Program ini tidak berfokus pada ideation atau pembuatan script dari nol, tetapi fokus pada tahap produksi visual: membaca storyboard dari JSON, menampilkan setiap scene, mengirim prompt image dan video ke provider lokal, lalu menyimpan serta menyediakan hasil image/video untuk di-preview dan di-download.
    13→    13→
    14→    14→Setiap scene memiliki tiga komponen utama:
    15→    15→
    16→    16→1. **VO** — narasi atau voice over.
    17→    17→2. **Image Prompt** — prompt untuk membuat gambar scene.
    18→    18→3. **Video Prompt** — prompt untuk membuat video dari gambar scene.
    19→    19→
    20→    20→Program harus mendukung aspect ratio `9:16` dan `16:9`, durasi total dari `30 detik` sampai `15 menit`, jumlah scene mengikuti isi JSON, serta tombol **Generate All** dengan sistem queue dan polling sehingga hasil scene yang selesai langsung muncul tanpa menunggu seluruh proses selesai.
    21→    21→
    22→    22→---
    23→    23→
    24→    24→## 2. Tujuan Produk
    25→    25→
    26→    26→Tujuan Kiwul Storyboard Studio:
    27→    27→
    28→    28→1. Membaca storyboard dari file JSON.
    29→    29→2. Menampilkan scene dalam UI yang mudah dipahami.
    30→    30→3. Menyediakan editor VO, image prompt, dan video prompt per scene.
    31→    31→4. Menghubungkan program ke provider image dan video.
    32→    32→5. Generate image per scene.
    33→    33→6. Generate video per scene.
    34→    34→7. Generate semua scene melalui tombol **Generate All**.
    35→    35→8. Memakai sistem queue dan polling.
    36→    36→9. Menampilkan hasil image/video langsung saat selesai.
    37→    37→10. Menyediakan tombol download image dan video.
    38→    38→11. Menyediakan export JSON final berisi path hasil generate.
    39→    39→12. Tidak hardcoded pada provider, model, aspect ratio, durasi, atau jumlah scene.
    40→    40→

    42→    42→
    43→    43→## 3. Masalah yang Diselesaikan
    44→    44→
    45→    45→Masalah utama dalam produksi video AI lokal:
    46→    46→
    47→    47→1. Storyboard sering hanya berupa teks atau JSON yang belum bisa langsung dieksekusi.
    48→    48→2. VO, image prompt, dan video prompt sering tidak sinkron.
    49→    49→3. User harus copy-paste prompt satu per satu ke engine image/video.
    50→    50→4. Hasil image/video sulit dikaitkan kembali ke scene.
    51→    51→5. Tidak ada status per scene.
    52→    52→6. Jika satu scene gagal, user sulit regenerate hanya scene itu.
    53→    53→7. Tidak ada tombol download per scene.
    54→    54→8. Tidak ada sistem batch generation yang aman untuk GPU lokal.
    55→    55→9. Tidak ada polling progress sehingga user tidak tahu proses sedang berjalan atau macet.
    56→    56→
    57→    57→Kiwul Storyboard Studio menyelesaikan masalah ini dengan satu workspace produksi storyboard yang terstruktur.

   594→   594→  "type": "generate_all",

   601→   601→
   602→   602→Setiap task:
   603→   603→
   604→   604→```json
   605→   605→{

  1202→  1202→
  1203→  1203→  project         StoryboardProject @relation(fields: [project_id], references: [id], onDelete: Cascade)
  1204→  1204→  tasks           StoryboardTask[]
  1205→  1205→}
  1206→  1206→
  1207→  1207→model StoryboardTask {
  1208→  1208→  id            String   @id @default(cuid())
  1209→  1209→  job_id        String
  1210→  1210→  scene_id      String
  1211→  1211→  task_type     String
  1212→  1212→  status        String   @default("queued")
  1213→  1213→  output_path   String?
  1214→  1214→  error_message String?
  1215→  1215→  started_at    DateTime?
  1216→  1216→  completed_at  DateTime?
  1217→  1217→  created_at    DateTime @default(now())
  1218→  1218→  updated_at    DateTime @updatedAt
  1219→  1219→
  1220→  1220→  job StoryboardJob @relation(fields: [job_id], references: [id], onDelete: Cascade)
  1221→  1221→}

  1226→  1226→## 32. Validasi JSON
  1227→  1227→
  1228→  1228→Saat load JSON, sistem wajib mengecek:
  1229→  1229→
  1230→  1230→1. JSON valid.
  1231→  1231→2. `project` ada.
  1232→  1232→3. `scenes` ada dan berupa array.
  1233→  1233→4. Minimal 1 scene.
  1234→  1234→5. Setiap scene punya `scene_id`.
  1235→  1235→6. Setiap scene punya `vo`.
  1236→  1236→7. Setiap scene punya `image_prompt`.
  1237→  1237→8. Setiap scene punya `video_prompt`.
  1238→  1238→9. Setiap scene punya `duration`.
  1239→  1239→10. Tidak ada `scene_id` duplikat.
  1240→  1240→11. `aspect_ratio` hanya `9:16` atau `16:9`.
  1241→  1241→12. Total durasi antara `30` dan `900` detik.
  1242→  1242→13. Jika `duration_seconds` tidak cocok dengan total scene, tampilkan warning.

  1373→  1373→## 38. Kesimpulan
  1374→  1374→
  1375→  1375→Kiwul Storyboard Studio adalah modul khusus untuk mengeksekusi storyboard dari JSON menjadi aset image dan video. Modul ini harus ringan, lokal, fleksibel, dan siap terhubung ke provider image/video seperti Z-Image, WAN, LTX, atau provider OpenAI-compatible lain.
  1376→  1376→
  1377→  1377→Fokus utama produk ini adalah:
  1378→  1378→
  1379→  1379→```text
  1380→  1380→Load JSON → Scene Editor → Provider Connection → Generate All → Polling → Preview → Download → Export JSON Final
  1381→  1381→```
  1382→  1382→
  1383→  1383→Produk ini menjadi fondasi penting untuk pipeline produksi video AI lokal karena memisahkan tahap storyboard dari tahap ideation, scriptwriting, TTS, subtitle, dan final render.
